# 初期移行候補 100件の選定と価格監査

## 1. 選定の全体フロー

```
active_listings_target.csv（6,145件）
        │
        │  ① 選定フィルタ（価格帯・需要指標・除外条件）
        ▼
   候補プール（約200件）
        │
        │  ② GetItem で詳細補完（category, condition, brand, images）
        ▼
   補完済み候補（約200件）
        │
        │  ③ price-auditor 監査（分類 A / B / C に仕分け）
        ▼
   ┌────┴────┬──────────────┐
   ▼         ▼              ▼
 分類 A    分類 B          分類 C
 自動変換   要確認          除外
   │         │              │
   │    原田に確認      初期移行から外す
   │    リスト提出
   │         │
   ▼         ▼
   ドラフト100件を選定
```


## 2. 初期移行候補100件の選定条件

### なぜ100件か

- 最終目標は 300〜500 SKU だが、最初の100件で移行パイプライン全体を検証する
- 100件なら GetItem 補完（50秒）+ 手動レビューも現実的な負荷
- 問題なければ残りを段階的に追加する

### 候補プール抽出の条件

| 条件 | 値 | 理由 |
|------|-----|------|
| **価格帯** | $100〜$999 | $100 未満は利益が薄い。$1,000 以上は手動レビュー対象（分類 C） |
| **Watchers** | 3 以上（基本条件） | 需要がある証拠。0 の商品は Shopify でも売れにくい |
| **数量** | 1（中古一品物） | 在庫同期がシンプル。複数在庫品は同期テスト後に追加 |

**Watchers 条件の緩和について:**
watchers >= 3 を基本条件とするが、特定の価格帯やカテゴリで候補数が不足する場合は
watchers >= 1 まで段階的に緩和できる。スクリプト内で緩和の余地を持たせ、
緩和した場合はレポートにその旨を明記する。

**数量が空欄・不明値の場合の扱い:**
quantity が空欄・0・数値変換不能の場合は「要確認」（分類 B）に振り分ける。
明らかに不正な値（負の数など）は「除外」（分類 C）とする。

### この条件で絞り込める件数（推定）

```
$100〜$999 かつ watchers ≥ 3 = 約 3,640 件
  → ここから約200件を抽出 → 監査後に100件を選定
```

### 候補プールからの200件抽出方法

需要が高く、価格帯がバランスよく分散するように層化抽出する:

| 価格帯 | 母集団（推定） | 抽出件数 | 理由 |
|--------|----------:|--------:|------|
| $100〜$200 | ~1,000 | 60 | 売れやすい。回転率で利益を稼ぐ |
| $200〜$300 | ~900 | 50 | 中間帯。品揃えの厚みを出す |
| $300〜$500 | ~1,000 | 50 | 利益額が大きい。コレクター需要 |
| $500〜$999 | ~700 | 40 | 高額コレクター。慎重に選定 |
| **合計** | | **200** | |

各帯の中で **watchers の多い順** にソートし、上位から抽出する。


## 3. price-auditor の監査条件

### 監査に使うデータ

候補プール200件に対して GetItem で補完した後、以下のデータが揃う:

| データ | 用途 |
|--------|------|
| price | eBay 出品価格 |
| category_name | 商品カテゴリ（同カテゴリ内の価格比較に使う） |
| condition_name | Condition マッピングの入力 |
| brand | Vendor 判定の入力 |
| image_count | 画像枚数の品質チェック |
| title | 関税上乗せヒューリスティックの入力 |

### 分類ルール

#### 分類 A: 自動変換候補

以下の **すべて** を満たす商品:

```
1. 価格が $100〜$999（$1,000 未満）
2. 同カテゴリ内での価格偏差が ±2σ 以内（外れ値でない）
3. 画像が 1枚以上ある
4. Condition が判定可能（空欄でない）
5. 数量が 1
```

→ **Shopify 基準価格 = eBay 価格 × 0.91** を自動適用

#### 分類 B: 要確認候補（原田に確認リスト提出）

以下の **いずれか** に該当する商品:

```
1. 同カテゴリ内での価格偏差が ±2σ を超える
   → 関税上乗せ or 特殊な価格設定の可能性
2. タイトルに以下のキーワードを含む（関税上乗せの手がかり）
   - "import", "customs", "tax included", "DDP"
   → 意図的に関税を含めている可能性
3. 画像が 0枚
   → GetItem で取得できなかった → 出品状態に問題がある可能性
4. 数量が空欄・0・数値変換不能
   → 出品データに不備がある可能性
```

→ **確認リスト（CSV）を生成し、原田に提出**

#### 分類 C: 初期除外

以下の **いずれか** に該当する商品:

```
1. 価格が $1,000 以上
   → 高額品は個別価格設定が必要
2. Condition が "For Parts" / "_EXCLUDE_"
   → Shopify で販売すべきでない
3. 数量が負の数など明らかに不正な値
   → データ不備
```

→ **初期移行から除外。第二弾で対応**

### 分類 B の「価格偏差」の計算方法

```
1. GetItem で取得した category_name でグルーピングする
   （例: "Action Figures", "Video Games" など）

2. 各カテゴリ内で価格の平均と標準偏差を計算する
   - ただしカテゴリ内の商品数が 5件未満の場合はスキップ
     （統計的に不十分）

3. 各商品の z-score を計算する
   z = (商品価格 - カテゴリ平均) / カテゴリ標準偏差

4. |z| > 2.0 なら「価格外れ値」→ 分類 B
```

**注意:** これは関税上乗せの「確実な検出」ではなく「疑いのフラグ付け」。
最終判断は原田が行う。


## 4. 原田が確認する内容

### 確認リストのフォーマット（CSV）

| 列 | 内容 | 入力者 |
|----|------|--------|
| item_id | eBay 商品ID | 自動 |
| sku | SKU | 自動 |
| title | 商品タイトル | 自動 |
| ebay_price | eBay 出品価格 | 自動 |
| shopify_price_draft | 暫定 Shopify 価格（× 0.91） | 自動 |
| category | eBay カテゴリ | 自動 |
| flag_reason | フラグ理由（価格外れ値 / キーワード / 画像なし） | 自動 |
| **decision** | **原田の判断** | **手動** |
| **custom_price** | **個別 Shopify 価格（任意）** | **手動** |

### decision 列の選択肢

| 値 | 意味 | 処理 |
|----|------|------|
| `ok` | 問題なし。× 0.91 で変換してよい | 分類 A に移動 |
| `exclude` | 初期移行から除外 | 分類 C に移動 |
| `custom` | 個別価格を指定する | custom_price 列の値を使用 |

### 確認フローの所要時間（想定）

- 分類 B は200件中 **10〜30件** と想定（外れ値 + キーワード該当）
- 1件あたり30秒で判断 → **5〜15分で完了**


## 5. 次に作るべきスクリプト

### スクリプト: `select_initial_candidates.py`

```
入力:  active_listings_target.csv（6,145件）
出力:
  - candidates_200.csv          候補プール200件（GetItem 補完前）
  - candidates_200_enriched.csv 候補プール200件（GetItem 補完後）
  - auto_convert.csv            分類 A: 自動変換候補
  - review_list.csv             分類 B: 原田確認リスト
  - initial_excluded.csv        分類 C: 初期除外
  - selection_report.txt        選定レポート（件数・価格帯・監査結果サマリ）
```

### 想定される出力件数

| 分類 | 件数（推定） | 説明 |
|------|----------:|------|
| A: 自動変換 | 160〜180 | 大多数はここに入る |
| B: 要確認 | 10〜30 | 価格外れ値 + キーワード該当 |
| C: 除外 | 5〜10 | Condition 不良、画像なし等 |
| **合計** | **200** | |

分類 A から watchers 上位100件を選び、ドラフト100件とする。
分類 B から原田が `ok` 判断した分をドラフトに追加。


## 6. オーナーが確認すべきこと

### スクリプト実行前に決めること

| # | 決定事項 | 推奨 | 理由 |
|---|---------|------|------|
| 1 | **候補の価格帯を $100〜$999 でよいか** | Yes | $100 未満は利益薄、$1,000+ は手動レビュー |
| 2 | **watchers ≧ 3 の閾値でよいか** | Yes | 3,640件のプールがあり100件選定に十分 |
| 3 | **初回は quantity = 1 のみに絞るか** | Yes | 在庫同期リスクを最小化 |
| 4 | **価格外れ値の閾値 2σ でよいか** | Yes | 緩すぎず厳しすぎない |

### スクリプト実行後に確認すること

| # | 確認事項 |
|---|---------|
| 1 | **分類 B（要確認）リストの各商品について、関税上乗せの有無を判断する** |
| 2 | **ドラフト100件の顔ぶれを見て、Shopify に出したくない商品がないか確認する** |
| 3 | **暫定 Shopify 価格（× 0.91）を見て、利益率に問題がないか確認する** |
