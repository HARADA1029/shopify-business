# eBay 商品データ分析（初期移行の前提資料）

## 目的

eBay に出品中の約8,000 SKU のデータ構造と傾向を整理し、
初期移行（300〜500 SKU）の選定基準を定める。
store-structure.md の3軸分類（商品タイプ / フランチャイズ / メーカー）に
マッピングしやすい形で分析観点を整理する。

---

## 分析に必要な eBay データ項目

### 必須項目（移行判断・マッピングに直結）

| # | eBay フィールド | 用途 | Shopify マッピング先 |
|:-:|---|---|---|
| 1 | **Item Title** | フランチャイズ名・キャラクター名・メーカー名の抽出元 | Title, Tags |
| 2 | **eBay Category** (Primary) | 商品タイプの分類元 | Product Type |
| 3 | **Condition** | 状態表記の対応表作成 | Condition タグ |
| 4 | **Price (USD)** | 価格帯分布の把握、高額品の識別 | Price |
| 5 | **Item Specifics: Brand** | メーカー名の抽出 | Vendor |
| 6 | **Quantity Available** | 在庫=1（一点物）か複数在庫かの識別 | 在庫同期リスク判定 |
| 7 | **Image URLs** | 画像枚数・品質の把握 | Images |
| 8 | **Item ID / SKU** | 一意識別子。eBay↔Shopify の紐付けに必須 | SKU メタフィールド |

### 補助項目（分析精度を上げる）

| # | eBay フィールド | 用途 |
|:-:|---|---|
| 9 | **Sold Quantity / Sales History** | 売上実績による優先度判定 |
| 10 | **View Count** | 注目度の把握（売れてなくても需要があるか） |
| 11 | **Watchers** | コレクターの注目度 |
| 12 | **Listing Start Date** | 長期滞留品の識別 |
| 13 | **Item Specifics: Character** | キャラクター名（入力されていれば） |
| 14 | **Item Specifics: Type** | 商品タイプの補助情報 |
| 15 | **Shipping Weight** | 配送料金テーブル設計の参考 |
| 16 | **Description (HTML)** | 商品説明の変換可能性の把握 |

### データ取得方法（要確認）

| 方法 | 手間 | 精度 | 備考 |
|---|---|---|---|
| eBay Seller Hub の CSV エクスポート | 低 | 中 | 一括ダウンロード可能。ただし項目が限定的な場合がある |
| eBay Trading API / Inventory API | 高 | 高 | 全フィールド取得可能。開発者アカウントが必要 |
| File Exchange (bulk download) | 低 | 中 | 一括ダウンロード。フォーマットが固定 |
| 手動サンプリング（50件程度） | 低 | 低 | まず傾向を掴むための初手として有効 |

**推奨:** まず Seller Hub の CSV エクスポートまたは手動サンプリング（50件）で
データ構造と傾向を把握し、その後 API での一括取得を検討する。

## 分析観点と整理フレーム

### 観点1: 商品タイプの分布

eBay カテゴリと Item Specifics から、store-structure.md の6タイプにどう分布するかを把握する。

| Shopify 商品タイプ | 対応しそうな eBay カテゴリ（推定） | 確認ポイント |
|---|---|---|
| Action Figures | Action Figures, Anime & Manga > Action Figures | eBay カテゴリ ID の特定 |
| Scale Figures | Anime & Manga > Figures, Statues | Action Figures との境界ルール |
| Model Kits | Models & Kits, Gundam > Model Kits | ガンプラの分類先 |
| Plush & Soft Toys | Stuffed Animals, Plush Toys | |
| Vintage & Retro Toys | Vintage Toys, Pre-1990s | 年代の判定基準 |
| Other Collectibles | 上記に該当しないもの | どの程度の割合があるか |

**確認すべきこと:**
- eBay カテゴリが Shopify の6タイプに概ねマッピングできるか
- 1つの eBay カテゴリが複数の Shopify タイプにまたがるケースがあるか
- 「Other」に落ちる商品がどの程度あるか

### 観点2: フランチャイズの分布

Item Title から作品名を抽出し、SKU 数の多い上位フランチャイズを特定する。

| 確認ポイント | 方法 |
|---|---|
| 上位10フランチャイズとその SKU 数 | タイトルのキーワード頻度分析 |
| フランチャイズの売上貢献度 | 売上データとのクロス集計 |
| 5件未満のフランチャイズの割合 | コレクション化しない「Other」の規模感 |

**想定される上位フランチャイズ（仮説・eBay 売上データで検証）:**
Dragon Ball, One Piece, Naruto, Gundam, Demon Slayer,
Pokemon, Evangelion, Sailor Moon, My Hero Academia, Studio Ghibli

### 観点3: メーカーの分布

Item Specifics の Brand フィールド、またはタイトルからメーカー名を抽出する。

| 確認ポイント | 方法 |
|---|---|
| 上位5メーカーとその SKU 数 | Brand フィールドの集計 |
| Brand フィールドの入力率 | 空欄の場合はタイトルから推定が必要 |
| メーカー名の表記ゆれ | "Bandai" / "BANDAI" / "バンダイ" 等の正規化ルール |

### 観点4: 価格帯の分布

移行対象の選定と、高リスク商品の識別に使う。

| 価格帯 | 分類 | 移行判断 |
|---|---|---|
| $1〜$30 | 低価格帯 | 移行優先。二重販売時の損失が小さい |
| $31〜$100 | 中価格帯 | 移行優先。ボリュームゾーンの可能性が高い |
| $101〜$300 | 高価格帯 | 選択的に移行。二重販売リスクと利益のバランス |
| $301〜 | 超高額帯 | 初期移行からは原則除外（後述の注記参照） |

### 観点5: Condition 表記の実態

eBay 側の Condition 表記を集計し、Shopify 側の4段階（Mint / Near Mint / Good / Fair）
との対応表を作成する。

| 確認ポイント | 方法 |
|---|---|
| eBay で使用されている Condition 値の一覧 | Condition フィールドの集計 |
| 各 Condition 値の SKU 数分布 | 頻度集計 |
| Condition Description（自由記述）の傾向 | テキスト分析（箱なし、付属品欠品 等） |

**eBay 標準 Condition 値（推定）:**

| eBay Condition | Shopify Condition タグ（案） |
|---|---|
| New / Brand New | Mint |
| New Other / Open Box | Near Mint |
| Used / Pre-Owned | Good（デフォルト。詳細は Description で補足） |
| Used - Like New | Near Mint |
| Used - Very Good | Good |
| Used - Good | Good |
| Used - Acceptable | Fair |
| For Parts / Not Working | 初期移行から除外 |

※ 実際の eBay 出品データで使用されている Condition 値を確認して対応表を確定する。

### 観点6: 画像の状況

| 確認ポイント | 方法 |
|---|---|
| 1商品あたりの平均画像枚数 | Image URL 数の集計 |
| 画像3枚未満の商品の割合 | 画像不足品の規模感（商品ページ品質に影響） |
| 画像の解像度・品質の傾向 | サンプリングによる目視確認 |

## 初期移行 SKU の選定基準

### 移行優先度スコア（6点満点）

各商品に以下の基準でスコアを付け、上位 300〜500 SKU を初期移行対象とする。

| 基準 | 加点条件 | 点数 | 補足 |
|---|---|---|---|
| **売上実績（商品単位）** | 同一商品または近似商品（同キャラ・同メーカー・同タイプ）が過去6ヶ月で売れている | +2 | eBay の Sold History で確認。「近似」は同フランチャイズ×同商品タイプで判定 |
| **カテゴリ需要** | 所属カテゴリ（フランチャイズ×商品タイプ）の全体売上が上位30%以内 | +1 | 個別商品の売上がなくても、カテゴリとして需要があれば加点 |
| **注目度** | Watchers 3件以上 or View Count 上位20% | +1 | 売れてなくても需要の兆候がある |
| **画像品質** | 画像3枚以上 | +1 | 商品ページ品質の最低基準 |
| **データ完備** | Title, Category, Brand, Condition がすべて入力済み | +1 | 移行時の手動補完コストが低い |

**スコアの使い方:**
- 4点以上: 優先移行（初期バッチの中核）
- 2〜3点: 選択的に移行（枠が余れば含める）
- 1点以下: 初期移行から除外

### 移行除外基準（以下に該当する商品は初期移行から外す）

| 除外条件 | 理由 |
|---|---|
| **価格 $300 超の一点物（初期の仮基準）** | 二重販売時の損失が大きい（inventory-sync-options.md の方針）。ただしこの金額ラインはカテゴリや代替可能性によって調整可能。例えば $300 超でも代替品が市場に流通しているカテゴリなら移行対象に含める余地がある。サンプル分析後にオーナーと最終ラインを確定する |
| **代替困難な希少品・限定品** | 二重販売でキャンセルした場合に代替を用意できない。価格に関わらず除外を推奨 |
| **Condition が "For Parts / Not Working"** | Shopify ストアの品質イメージを損なう |
| **画像1枚以下** | 商品ページの品質基準を満たさない |
| **タイトルが日本語のみ** | 英語タイトルへの変換コストが高い。後続バッチで対応 |
| **eBay 出品から1年以上経過し View/Watcher ゼロ** | 需要が見込めない滞留品 |

### 初期移行に向く商品群（優先投入）

| 商品群 | 理由 |
|---|---|
| **中価格帯（$30〜$100）のアクションフィギュア** | eBay でのボリュームゾーンかつ、二重販売リスクが限定的 |
| **人気フランチャイズ（Dragon Ball, One Piece 等）のスケールフィギュア** | 検索需要が高く、Shopify でも集客しやすい |
| **Bandai / Good Smile 等の有名メーカー品** | メーカー名でフィルター検索するコレクターを獲得できる |
| **状態が良い商品（Mint / Near Mint）** | ストアの第一印象が良くなる。レビュー獲得にも有利 |
| **画像3枚以上の商品** | 商品ページの品質が担保できる |
| **売上実績のある商品と同カテゴリの商品** | 需要が検証済みのカテゴリ |

### 初期は外した方がよい商品群（後続バッチで対応）

| 商品群 | 理由 | 対応時期 |
|---|---|---|
| **高額一点物（仮基準 $300 超・代替困難）** | 二重販売リスクが高い | 在庫同期が安定してから |
| **希少限定品・プレミア品** | 代替困難。キャンセル時のダメージが大きい | 同上 |
| **Vintage & Retro（$200超）** | 高額 + 状態の判定が難しい | 運用ノウハウ蓄積後 |
| **日本語のみのタイトル** | 英語変換が必要。初期は変換済みの商品を優先 | 変換ルール確立後 |
| **画像1枚以下の商品** | 商品ページ品質基準を満たさない | 再撮影後 |
| **ジャンク品・パーツ品** | ストアの品質イメージを損なう | 当面は移行しない |

## 分析の進め方（推奨手順）

### Step 1: サンプリング（Day 4）
- eBay Seller Hub から CSV エクスポート、または手動で50件をサンプリング
- データ構造と入力状況を確認する
- 上記6観点のうち、どの項目が取得可能かを判定する

### Step 2: 傾向分析（Day 4〜5）
- 商品タイプ / フランチャイズ / メーカーの分布を大まかに把握
- 価格帯分布を把握
- Condition 表記の実態を確認し、対応表のドラフトを作成

### Step 3: 選定基準の適用（Day 5〜6）
- 移行優先度スコアを50件サンプルに試適用し、基準の妥当性を検証
- 除外基準に該当する商品の割合を概算
- 初期移行の300〜500 SKU が確保できるか見通しを立てる

### Step 4: 移行対象リストの確定（Day 6〜7）
- 全8,000 SKU に選定基準を適用（CSV or API で一括処理）
- 上位 300〜500 SKU を移行対象リストとして確定
- catalog-migration-planner に引き渡す
