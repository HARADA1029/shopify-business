# 在庫同期方式の比較（eBay ↔ Shopify）

## 目的

eBay と Shopify の在庫を同期し、二重販売を防止するための方式を比較・選定する。
本プロジェクト最大のリスク領域であるため、安全性を最優先に評価する。

---

## 前提条件（評価にあたっての制約）

| 項目 | 内容 |
|---|---|
| 商材の特性 | **中古の一点物が多い**。同一商品の複数在庫はほぼない。一点物は在庫=1のため、売れた瞬間に相手チャネルを停止しないと二重販売になる |
| eBay 出品数 | 約8,000 SKU |
| Shopify 初期移行数 | 300〜500 SKU（全商品は移行しない） |
| 段階 | 0→1 の立ち上げ。運用実績なし |
| 運営体制 | 個人運営（1名） |
| 技術力 | ebay-tools で Playwright ベースのツールを自作できるレベル |
| 予算 | 過剰投資を避けたい（月額 ¥5,000〜¥10,000 程度が目安） |

### 一点物における二重販売の深刻度

通常の在庫商品（在庫10個）なら、同期遅延で1個ずれても影響は小さい。
しかし **在庫=1 の一点物** では、同期遅延=二重販売 になる。

二重販売が発生した場合:
- 片方の購入者にキャンセル連絡 → 信用低下
- eBay ではセラー欠陥（defect）としてアカウント評価に影響
- 返金処理の手間とコスト

**この前提のため、同期の即時性と確実性が最も重要な評価軸になる。**

---

## 比較対象の4方式

### A. SaaS ツール（LitCommerce, Codisto, Sellbrite 等）

在庫同期専用の既存クラウドサービスを利用する。

| 項目 | 評価 |
|---|---|
| **同期速度** | 準リアルタイム（5〜15分間隔が一般的。ツールにより異なる） |
| **二重販売リスク** | **中〜高**。一点物で5〜15分の遅延は危険。eBay で売れた直後に Shopify で売れる可能性がある |
| **初期構築の手間** | 低。GUI で設定するだけ |
| **月額コスト** | ¥3,000〜¥15,000（SKU数・機能による） |
| **カスタマイズ性** | 低〜中。ツール側の仕様に依存 |
| **信頼性** | ツール側の障害・仕様変更に依存。自分でコントロールできない |
| **スケーラビリティ** | 高。8,000 SKU 全量でも対応可能 |

**代表的なツール:**

| ツール | 月額目安 | 同期間隔 | 備考 |
|---|---|---|---|
| LitCommerce | $29〜 | 15分〜 | eBay↔Shopify 対応。UI がわかりやすい |
| Codisto (by Linnworks) | $29〜 | 準リアルタイム | eBay 連携に強い。Shopify アプリとして動作 |
| Sellbrite | $19〜 | 15分〜 | 複数チャネル対応。GoDaddy 傘下 |
| CedCommerce | $19〜 | 可変 | Shopify アプリとして動作 |

※ 同期間隔・機能は各ツールの公式情報で最新を確認すること。

---

### B. 半手動運用（スプレッドシート + 手動更新）

Shopify 移行した商品の在庫管理を、スプレッドシートで台帳管理し、
eBay/Shopify のどちらかで売れたら手動でもう片方を停止する。

| 項目 | 評価 |
|---|---|
| **同期速度** | 手動のため数分〜数時間の遅延 |
| **二重販売リスク** | **高**。手動対応の遅延・見落としが確実に発生する |
| **初期構築の手間** | 低。スプレッドシートを作るだけ |
| **月額コスト** | ¥0 |
| **カスタマイズ性** | 高。自由に設計できる |
| **信頼性** | 低。人的ミスに依存 |
| **スケーラビリティ** | 低。100 SKU 程度が限界 |

---

### C. 自前実装（eBay API + Shopify API）

eBay API と Shopify API を自前でつなぎ、在庫変動を自動同期するシステムを構築する。

| 項目 | 評価 |
|---|---|
| **同期速度** | リアルタイム可能（Webhook + API で数秒以内） |
| **二重販売リスク** | **低**。Webhook で即時検知し、API で即時停止できる |
| **初期構築の手間** | **高**。eBay API の認証・Webhook 設定、Shopify API 連携、エラーハンドリング、監視、すべて自前で構築 |
| **月額コスト** | サーバー費のみ（¥1,000〜¥3,000） |
| **カスタマイズ性** | 最高。完全に自分の要件に合わせられる |
| **信頼性** | 自分のコード品質次第。バグ=二重販売 |
| **スケーラビリティ** | 設計次第で高い |

**技術的な課題:**
- eBay Notification API（Webhook）の設定と受信サーバーが必要
- Shopify Webhook の設定と受信処理
- 両方の API のレート制限への対応
- エラー時のリトライ・デッドレター処理
- 24時間稼働のサーバー運用

---

### D. ハイブリッド（SaaS + 運用セーフガード）

SaaS ツールで基本の同期を行いつつ、
移行対象の選定と運用ルールでリスクを制御する。

| 項目 | 評価 |
|---|---|
| **同期速度** | SaaS の同期速度（5〜15分）+ 運用ルールで補完 |
| **二重販売リスク** | **低〜中**。高リスク SKU を移行対象から外し、運用ルールで残存リスクをカバー |
| **初期構築の手間** | 低〜中。SaaS 設定 + 運用ルール策定 |
| **月額コスト** | SaaS 費用（¥3,000〜¥15,000） |
| **カスタマイズ性** | 中 |
| **信頼性** | SaaS + 運用ルールの二重構造 |
| **スケーラビリティ** | 中。SKU 拡大時は運用負荷が増える |

**運用セーフガード:**
- **高額一点物・代替困難な商品は初期移行対象から外す**（事故確率を元から下げる）
- **Shopify 注文発生直後に、最優先で eBay 側の出品停止を確認する**（通知設定 + 即時対応）
- **日次で同期ズレがないかスプレッドシートで突合チェック**（SaaS の見落としを検出）

## 推奨案

### 推奨: D. ハイブリッド（SaaS + 運用セーフガード）で開始し、将来的に C. 自前実装に移行

**フェーズ1（今〜テスト移行期）: ハイブリッド**

- SaaS ツール（LitCommerce or Codisto）で 300〜500 SKU の基本同期を構築する
- 高額一点物・代替困難な商品は初期の移行対象から外す
- Shopify の注文通知を即時受信し、注文発生直後に eBay 側の出品停止を確認する
- 日次でスプレッドシートによる同期ズレ突合チェックを実施する

**フェーズ2（月商安定後）: 自前実装への移行を検討**

- SaaS の同期遅延による二重販売が実際に発生した場合、自前実装を開発する
- eBay API Webhook + Shopify Webhook によるリアルタイム同期
- 移行判断基準: 月間の二重販売件数が1件以上発生、または月商100万円超で SaaS コストが割に合わない場合

### この推奨案の理由

1. **0→1 段階で自前実装は過剰投資** — eBay API の Webhook 設定、受信サーバー構築、エラーハンドリング、24時間運用は、まだ売上がない段階で着手するにはリスクとコストが大きい。まず SaaS で動かし、実際の同期問題を体験してから自前開発の要件を固める方が合理的

2. **高リスク SKU を初期移行対象から外すことで事故確率を下げる** — 高額一点物や代替困難な商品を移行対象から除外すれば、万一の二重販売でも被害が限定的になる。初期移行の 300〜500 SKU は「二重販売が起きても対処可能な商品」から選定する

3. **SaaS 単体では一点物のリスクをカバーしきれない** — 5〜15分の同期遅延は、在庫=1 の一点物には危険。しかし注文直後の即時確認ルールと日次突合で残存リスクを大幅に低減できる

4. **300〜500 SKU なら運用セーフガードが現実的** — 8,000 SKU なら日次突合は破綻するが、初期の限定移行なら負荷は許容範囲内。この段階でオペレーションのノウハウを蓄積できる

5. **SaaS の実運用データが自前実装の要件定義になる** — SaaS を使うことで「どのタイミングで同期ズレが起きるか」「どの商品カテゴリでリスクが高いか」の実データが得られる。これが将来の自前実装の最良の要件定義になる

6. **立ち上げ速度を最優先できる** — SaaS なら数日で同期を開始できるため、14日計画のスケジュールに収まる

---

## SaaS ツールの選定方針

推奨案を採用する場合、次のステップで SaaS ツールを選定する。

### 選定基準（優先順）

1. **eBay ↔ Shopify の在庫同期に対応** しているか
2. **同期間隔が短い**（15分以下、できれば5分以下）か
3. **日本の eBay セラーアカウントに対応** しているか
4. **300〜500 SKU の価格帯** が月額 ¥5,000 以下か
5. 無料トライアルがあるか

### 次のアクション

- LitCommerce / Codisto / CedCommerce の3候補で、上記基準を満たすか個別調査する
- 無料トライアルで実際に eBay アカウントを接続し、同期動作を検証する
- テスト移行の 10〜20 SKU で同期遅延を計測する
