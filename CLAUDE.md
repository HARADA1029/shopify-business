# Shopify Business Project

## プロジェクト概要

日本から全世界へ中古フィギュア・おもちゃを輸出販売する Shopify ストアの新規立ち上げ。
既存の eBay 事業（月間利益約100万円、約8000商品出品中）を基盤として、
Shopify を新たな販売チャネルとして追加する。

## 前提条件

- eBay アカウント: 稼働中、約8000 SKU 出品済み
- Shopify: 0→1 の新規立ち上げ
- 対象商材: 中古の日本フィギュア、おもちゃ系が中心
- 対象市場: 英語圏中心で開始（US, UK, AU, CA）、将来的に拡張
- 初期移行: 300〜500 SKU のテスト移行から開始（全商品移行はしない）
- 在庫管理: eBay と Shopify の在庫同期が最重要リスク

## 最重要ルール: eBay 既存事業の保護

**本プロジェクトでは、eBay 既存アカウント・既存出品に変更を加えない。**

- 使用する eBay API は **読み取り専用のみ**（GetMyeBaySelling / GetItem 等）
- **ReviseItem / EndItem / AddItem** などの書き込み系 API は使用しない
- eBay の listing / description / images / messages に **Shopify の URL やストア名を記載しない**（eBay の off-eBay 誘導ポリシー違反リスク）
- Shopify は eBay と **完全に独立した追加チャネル** として構築する
- **eBay の既存売上を守ることを最優先** とする
- 将来的に在庫同期を導入する際も、方式決定・テスト検証を経てから慎重に進める。eBay 側への書き込みは原田の事前承認が必須

## 最重要リスク

**在庫同期ミス** — 同一商品を eBay と Shopify の両方で販売するため、
片方で売れた商品がもう片方で売れてしまう「二重販売」を防ぐ仕組みが必須。
設計・実装のすべてのフェーズでこのリスクを最優先で考慮すること。

## フォルダ構成

```
shopify-business/
├── CLAUDE.md                 # このファイル（プロジェクトルール）
├── docs/                     # 設計・計画ドキュメント
├── store/                    # Shopify ストア（テーマ・設定）
│   ├── theme/                # テーマカスタマイズ
│   └── config/               # ストア設定エクスポート
├── inventory-sync/           # 在庫同期システム（最重要）
│   ├── src/                  # ソースコード
│   └── tests/                # テスト
├── product-migration/        # eBay → Shopify 商品データ移行
│   ├── scripts/              # 移行スクリプト
│   ├── mappings/             # カテゴリ・属性マッピング
│   └── data/                 # 移行データ（.gitignore 対象）
├── marketing/                # マーケティング
│   ├── seo/                  # SEO 設定
│   └── ads/                  # 広告設定
└── ops/                      # 運用
    ├── shipping/             # 配送設定・料金テーブル
    └── monitoring/           # 在庫同期監視・アラート
```

## 開発ルール

- コメントは日本語で書く
- ドキュメントは日本語で書く（ストアの顧客向けコンテンツは英語）
- ebay-tools/ とは独立して管理する（相互参照しない）
- product-migration/data/ には個人情報や API キーを含むファイルを置かない
- 在庫同期に関する変更は必ずテストを書いてから実装する
- 重要な意思決定は docs/ に記録する
